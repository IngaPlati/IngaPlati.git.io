## Reflection on Coding My CV Using Markdown 

### Situation
I was required to create my CV using Markdown for the first time as part of my digital portfolio. I was used to using Microsoft Word, where formatting is visual and immediate, so working in a plain-text environment was unfamiliar.

### Task
My task was to structure and format a professional CV using Markdown that would be clear, readable, and suitable for publishing online.

### Action
I learned basic Markdown syntax such as headings, bullet points, and text formatting. I struggled with spacing, line breaks, and maintaining a consistent layout because Markdown does not provide full control over design like Word. Small mistakes in syntax sometimes caused formatting issues, and it was not always obvious what went wrong.

To overcome this, I used trial and error, repeatedly previewing my work and adjusting the syntax. I also simplified my layout to match what Markdown can handle instead of forcing complex formatting.

### Result
I successfully created a clean and structured CV. I improved my understanding of how developers format content for the web and developed stronger attention to detail. This experience also improved my ability to troubleshoot and adapt to new tools.

### Situation
After completing my CV, I needed to embed a mock interview video into the document using Markdown or HTML.

### Task
My task was to ensure that the video could be viewed directly within my CV when accessed online.

### Action

### Result
I successfully embedded the video into my CV. This helped me understand how Markdown can be extended using HTML and improved my ability to debug and fix technical issues.

### Action
